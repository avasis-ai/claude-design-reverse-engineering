# Patterns

Each pattern note should be a standalone markdown file with:

1. **Name** — short identifier (`chunking-lists`, `comparison-table`, …).
2. **Observation** — what readers or users experience.
3. **When to use** — job-to-be-done.
4. **Anti-patterns** — known failure modes.
5. **Examples** — sanitized snippets or links to public pages.

Open a PR to add `patterns/<slug>.md`.
